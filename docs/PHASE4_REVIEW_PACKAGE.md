## 1. MedIntel の目的

```
MedIntel = 地域医療の不一致・構造パターンを発見する仮説生成装置
         ≠ 診断ツール
         ≠ ランキングサイト
         ≠ 政策判断ツール
```

**何を発見できるか**:
- 47都道府県スケールでの「リスク・受療・利用・供給・結果」の **6軸独立評価**
- これらの軸の **不一致または特徴的な構造プロファイル**
- 6つの地域医療プロファイル (multi-label archetype) への分類による仮説生成

**何を主張しないか**:
- 因果推論しない (「Xを増やせばYが下がる」という主張はしない)
- 医療の優劣を断定しない (「支援型 = 良い医療」とは言わない)
- 個別施設の評価をしない
- 政策推奨をしない
- 患者個人の予後を予測しない

## 3. 6軸モデル (Bridge Risk Model v1)

各軸は **独立に評価** する。線形連鎖や因果は仮定しない。

```
┌──────────────────────────────────────────────────────┐
│ 6 independent axes (順序は読む方向、因果ではない)         │
├──────────────────────────────────────────────────────┤
│ Risk      | NDB特定健診 (HbA1c/BMI/SBP/LDL/尿蛋白) +     │
│           |  質問票14項目 (喫煙・服薬・既往)              │
│ Demand    | 患者調査受療率 (大分類14項目, 10万対)          │
│ Use       | NDB処方 (薬効分類別) + NDB診療行為 (3区分)    │
│ Supply    | 病床機能報告 (4区分) + 施設基準 cap proxy     │
│ Outcome   | 死亡率 粗 + 年齢調整死亡率 (2020)             │
│ Aging     | 住基 (75歳以上割合) + 社人研推計             │
└──────────────────────────────────────────────────────┘
```

**注**: 各軸は独立軸であり、矢印で結ばれた線形連鎖は **仮定しない**。 「Risk 高 → Outcome 悪」「Supply 高 → Outcome 良」という単純な推論は **本ツールの主張ではない**。

### 3.1 6疾患領域 (Bridge)
- 循環器 (心疾患・高血圧・脂質異常・喫煙)
- 糖尿病・代謝 (HbA1c・BMI・糖尿病薬)
- がん (悪性新生物・喫煙・飲酒)
- 脳血管 (脳卒中既往・血圧)
- 呼吸器 (喫煙・気管支拡張剤)
- 腎疾患 (尿蛋白・eGFR・CKD既往)

## 4. 6つの地域医療プロファイル (Phase 4-0 で統合) ⚠️ multi-label archetype の二層構造

### 4.1 重要な前提 (peer review v1 採択)

> これら6つは **排他的分類ではなく**、都道府県が複数プロファイルに同時該当しうる **multi-label archetype** である。  
> **MECE な分類体系ではなく、仮説生成のための観察ラベル** である。  
>
> 例: 秋田県は Pattern 2 (Supply-Outcome 並列悪化) と Pattern 5 (高齢化-在宅移行ギャップ) の **両方に該当** する。

### 4.2 二層構造 (peer review v2 採択)

reviewer指摘に基づき、6 archetype を **2 層** に分離:

#### A. Mismatch Signal Tags (不一致シグナル) — 仮説生成の主対象

| # | プロファイル | 内容 | 軸 | 代表県 | 関連 docs |
|---|---|---|---|---|---|
| **1** | **Risk-Care Gap** | リスク高 × 受療低 | Risk vs Demand | **沖縄** (糖尿病) | OKINAWA_DIABETES_PARADOX.md |
| **2** | **Supply-Outcome Gap (並列悪化)** | 供給薄 × 結果悪 | Supply vs Outcome | **秋田・青森・岩手・山形** | PHASE_2E_2_TOHOKU_HOMECARE.md |
| **3** | **Supply-Outcome Mismatch** | 供給+ × 結果悪 | Supply vs Outcome | **山口・徳島・鹿児島** | PHASE_2E_3_WESTERN_JAPAN_HOMECARE.md |
| **5** | **Aging-Outcome Burden** | 高齢化進行 × 在宅薄 | Aging vs Supply | **秋田・青森・北海道(特殊)** | PHASE_2E_2_TOHOKU_HOMECARE.md |

#### B. Context Archetypes (背景構造プロファイル) — 単独では不一致を示さない

| # | プロファイル | 内容 | 軸 | 代表県 | 関連 docs |
|---|---|---|---|---|---|
| **4** | **Supply-Outcome Alignment Context** | 供給+ × 結果良 | Supply vs Outcome | **岡山・熊本・島根** | PHASE_2E_3_WESTERN_JAPAN_HOMECARE.md |
| **6** | **Urban Low-risk / High-capability Context** | 大都市 × 高機能集積 | Risk + Supply | **東京・大阪** | (Phase 1〜2 全体) |

**重要**: Pattern 4 と 6 は **Mismatch Signal ではなく Context Archetype**。背景構造の参照ラベルとして使用し、単独では仮説生成の主対象とはしない。

### 4.3 軸の対応関係 (multi-label の検証)

```
Mismatch Signal Tags:
  Risk vs Demand   軸 → Pattern 1 (Risk-Care Gap)
  Supply vs Outcome 軸 → Pattern 2 (Gap), Pattern 3 (Mismatch)
  Aging vs Supply  軸 → Pattern 5 (Aging-Outcome Burden)

Context Archetypes:
  Supply vs Outcome 軸 → Pattern 4 (Alignment Context)
  Risk + Supply 結合 → Pattern 6 (Urban Context)
```

→ Pattern 2 と 5 は秋田・青森で **重複** し得る。これは multi-label 仕様であって不具合ではない。  
→ 同一県が Mismatch Signal と Context Archetype の両方に該当することも有り得る。

## 5. 代表例 (4地域の発見) ⚠️ 実値 vs 全国比 を分離

### 5.1 沖縄県 (Pattern 1: Risk-Care 乖離)

**主張**: リスクは47県中上位だが、医療接触 (受療率・処方) は中位以下

| 指標 | 沖縄値 (実値) | 47県順位 | 全国比 |
|---|---|---|---|
| BMI≥25 | **39.8%** | **1位** (最高) | 47県平均比 +32.3% |
| HbA1c≥6.5 | **8.7%** | 6位 | 47県平均比 +12.9% |
| 内分泌・代謝系外来受療率 | **184/10万** | **47位** (最低) | 全国比 -47.1% |
| 糖尿病薬服用率 | **6.9%** | 12位 (高位) | 47県平均比 +8.2% |
| 糖尿病粗死亡率 | 11.0/10万 | 40位 (低位) | -11.3% |
| 糖尿病年齢調整死亡率 (男, 2020) | **20.8/10万** | **2位** (高位) | +49.6% |

**逆転現象**: 粗死亡率では下位40位だが、年齢調整 (2020) では男女とも2位。

⚠️ **時点差の留保**: 年齢調整死亡率は **2020年 (令和2年)** データ。NDB特定健診 (令和4年) ・NDB処方 (令和5年) との **時点差は2-3年**。同時点の因果関係ではなく、**地域構造の補助アウトカム指標** として扱う。

**慎重な仮説**: 仮説F (中年男性早期死亡で高齢糖尿病患者まで生存しない) を **支持しうる重要な補足所見**。ただし時点差・コーディング差・受診者選択バイアス等の他仮説 (A〜E) も並列の説明力を持ち、本データだけで因果は推論できない。

### 5.2 秋田県 (Pattern 2 + 5: 並列悪化 + 高齢化ギャップ、multi-label例)

**主張**: 高齢化最高水準なのに在宅医療 proxy が薄く、結果指標も悪い (2プロファイル該当)

| 指標 | 秋田値 (実値) | 全国比 |
|---|---|---|
| 75歳以上割合 | 22.0% | **+4.1pt** (47県中3位) |
| NDB在宅 (75+10万対) | 173,533 | **-47%** |
| 慢性期病床 (75+10万対) | 1,085 | -28% |
| cap.homecare (75+10万対) | 187 | **-44%** |
| cap.rehab (75+10万対) | 43.0 | **-43%** |
| 脳血管疾患 年齢調整死亡率 (2020) | 101.2/10万 | **+35%** |

⚠️ **時点差の留保**: 年齢調整死亡率は2020年データ。住基 (2025) ・病床機能 (2024) ・cap proxy (2026年2-4月) との **時点差は3-5年**。

**5指標すべて -28%以上**で在宅移行 proxy が薄く、脳血管調整死亡率も全国比+35%。Pattern 2 (Supply-Outcome 並列悪化) と Pattern 5 (高齢化-在宅移行ギャップ) の **両方に該当する multi-label 例**。

### 5.3 山口県 (Pattern 3: Supply-Outcome 不一致 — 重要な反例)

**主張**: 在宅移行 proxy が47県内でも厚いが、結果指標 (肺炎・心疾患) は悪い

| 指標 | 山口値 (実値) | 全国比 |
|---|---|---|
| 75歳以上割合 | 20.9% | +3.0pt |
| cap.homecare (75+10万対) | 679.5 | **+104%** |
| cap.rehab (75+10万対) | 135.8 | **+81%** |
| 慢性期病床 (75+10万対) | 2,342 | +55% |
| 肺炎 年齢調整死亡率 (2020) | 81.0/10万 | **+31%** |
| 心疾患 年齢調整死亡率 (2020) | 171.2/10万 | **+14%** |
| 腎不全 年齢調整死亡率 (2020) | 23.9/10万 | **+17%** |

⚠️ **時点差の留保**: 同上 (2020 vs 2026年4月の cap proxy)。

**「供給 proxy が厚いにもかかわらず、結果指標に高値傾向が見られる」** 代表例。施設基準届出件数は47県内でも最厚レベルだが、肺炎・心疾患・腎不全は悪化方向。

⚠️ **避けるべき表現**: 「供給が効いていない」「在宅医療が有効でない」「医療体制が悪い」は **誤った因果推論**。

### 5.4 東京都 (Pattern 6: 都市低リスク・高機能集積 — 構造プロファイル)

**主張**: 若年人口比率が高く、リスクは低位、Tier S/A 病院が集積

| 指標 | 東京値 (実値) | 47県順位 |
|---|---|---|
| 75歳以上割合 | 13.9% | **47位** (最低) |
| HbA1c≥6.5 | **6.0%** | **47位** (最低) |
| BMI≥25 | 27.9% | 9位 (低位寄り) |
| Tier S 病院 (高機能) | 集積あり | 上位 |
| 腎不全 粗死亡率 | 12.9/10万 | **47位** (最低) |
| 糖尿病 年齢調整死亡率 (男, 2020) | 13.7 | 27位 (中位) |

⚠️ **時点差の留保**: 年齢調整死亡率は2020年データ。

**注釈の修正 (peer review v1 採択)**: 低リスク・低死亡率は **若年構造のみで説明できるとは限らない**。健診受診者選択バイアス、生活習慣、医療アクセス、都市部の人口構成、社会経済要因が **複合している可能性** がある。Pattern 6 は「**不一致**」ではなく **「構造プロファイル」** に近い。

## 6. 誤読防止ルール (peer review遵守、必読)

### 6.1 ❌ してはいけない読み方

| 禁止表現 | 理由 |
|---|---|
| 「県Xは医療の質が低い (or 高い)」 | データから優劣は推論不可 |
| 「在宅医療 cap が厚い → 結果が良い」 | 山口/徳島/鹿児島で反証 |
| 「リスク高 → 受療高 → 結果改善」 | 沖縄で反証 |
| 「供給が効いていない」「在宅医療が有効でない」 | 因果推論であり data から導けない |
| 「県Xは政策Yを実施すべき」 | 政策推奨ではない |
| 「Tier C/D = 機能が低い」 | Tier は規模・実績の参考指標 |
| 「ランキング順位が高い = 優れた県」 | 単一指標評価は誤り |
| 「6プロファイルは MECE 分類」 | multi-label であり MECE ではない |

### 6.2 ✅ 推奨される読み方

| 推奨表現 | 意味 |
|---|---|
| 「リスク proxy が高い」 | 健診階級分布で識別 |
| 「供給 proxy が厚い (薄い)」 | 施設基準届出件数の集計 |
| 「結果指標が全国平均と並列に悪化」 | 因果は不明だが事実として記述 |
| 「~が見られる」 | 観察の記述 |
| 「不一致パターン/構造プロファイルを示す」 | 発見であって診断ではない |
| 「追加調査の動機付け」 | 行動推奨ではない |
| 「複数プロファイルに同時該当しうる」 | multi-label性の明示 |
| 「年齢調整死亡率は2020年、他は2022-2024年」 | 時点ズレ明示 |

### 6.3 UI用の短縮ステートメント (将来UI実装時に必須)

> この分類は地域医療の **仮説生成用** です。  
> 医療の質・政策効果・施設の優劣を判定するものではありません。  
> 6プロファイルは排他的分類ではなく、複数該当しうる observational labels です。

- (案C) 各軸の数値だけ表示し、パターン分類は表示しない

### Q5. 年齢調整死亡率の Bridge 反映
Bridge OUTCOME 列に年齢調整死亡率 (2020) を入れるべきか?  
- 案A: 粗死亡率 (2024) を主表示 + 年齢調整 (2020) を紫色併記
- 案B: 2行表示 (粗 / 年齢調整) ← peer review v1 推奨
- 案C: トグル切替
- そもそも入れない方が安全か?

### Q6. 供給 proxy と結果指標の並列の危険な誤読
山口県の「供給+ × 結果悪」を Bridge 1画面に並べたとき、最も危険な誤読は何か?
- (例) 「在宅医療を増やしても効果がない」
- (例) 「医療体制が良いのに患者が多い = 県民の健康意識が低い」
